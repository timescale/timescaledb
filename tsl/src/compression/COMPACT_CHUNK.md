# compact_chunk

Merges overlapping compressed batches within a chunk. Only touches batches
that need fixing вҖ” correctly ordered batches are left as-is.

## How It Works

```
 Phase 1: FIND          Phase 2: RECOMPRESS       Phase 3: VERIFY
 в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ       в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ      в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ

 в”Ӯ Stop at      в”Ӯ       в”Ӯ overlapping      в”Ӯ      в”Ӯ fresh snapshot в”Ӯ
 в”Ӯ first issue  в”Ӯ       в”Ӯ batches, continueв”Ӯ      в”Ӯ Clear UNORDEREDв”Ӯ
 в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ       в”Ӯ scanning for moreв”Ӯ      в”Ӯ if clean       в”Ӯ
                        в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ      в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

### 5. Multi-column orderby вҖ” boundary tie resolution

When col1 min/max ranges touch, decompress actual boundary rows to check secondary columns.

```
 orderby='device,time'     Both batches have device min=d2, max=d2 (col1 tie)

 Batch 1: (d1..d2, 00:01..08:20)    last row:  (d2, 08:20)  в—„в”Җ decompress
 Batch 2: (d2..d3, 08:00..16:40)    first row: (d2, 08:21)  в—„в”Җ decompress
                                     08:20 < 08:21 вҶ’ no overlap вң“

 Batch 1: (d1..d2, 00:01..08:20)    last row:  (d2, 08:20)
 Batch 2: (d2..d3, 04:11..12:30)    first row: (d2, 08:11)
                                     08:20 > 08:11 вҶ’ OVERLAP вҶ’ merge
```

### 6. Mixed-null batch splitting

A batch with both NULL and non-NULL values in the first orderby column is split вҖ” NULLs are invisible to the index and would be unordered.

```
 orderby='value NULLS LAST'

 в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ  в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ      в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ  в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ  в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”җ
 в”Ӯ 1001..1800, NULLГ—200в”Ӯ  в”Ӯ1801..2800в”Ӯ в”Җв”Җв–¶  в”Ӯ1001..1800в”Ӯ  в”Ӯ1801..2800в”Ӯ  в”ӮNULL  в”Ӯ
 в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ  в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ      в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ  в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ  в”Ӯ Г—200 в”Ӯ
  в–І index says min=1001                     split out в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв–¶в””в”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
    NULLs invisible                         non-null kept   untouched   end (NULLS LAST)
```

### 7. Overlap merge preserving NULLs

When overlapping batches with nullable first orderby are merged, NULL rows are flushed as pure-null batches.

    в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ        overlap                                 в””в”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

### 8. Secondary column NULLs at boundary tie

Boundary row comparison respects NULLS FIRST/LAST semantics instead of skipping NULLs вҖ” skipping would leave the batches unordered.

```
 orderby='time, value NULLS LAST'

 Batch 1 last row:  (08:20, NULL)         CORRECT: NULL with NULLS LAST
 Batch 2 first row: (08:20, 1001)         means NULL > 1001 вҶ’ OVERLAP вҶ’ merge
